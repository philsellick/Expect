Type Definitions

AllEventsCallback(eventObject)

The AllEventsCallback is used when subscribing to every event on a given channel. The callback takes an 'event' parameter object that contains the event name and the event payload

Name Type Description
eventObject Object

event object received from push server

Name Type Description
event string

name of the event
payload EventPayload

payload from the event

APIErrorCallback(error)

The APIErrorCallback handles unsuccessful response from the API. Every error response from the api conforms to the same format

Name Type Description
error Object

error object containing information about an API Error

Name Type Description
code number

API error code
message string

API error message
type string

API error type

APISuccessCallback(result)

The APISuccessCallback handles successful responses from the API. Every response from the MindMeld API conforms to the same format

Name Type Description
result Object

result object containing response from the API

Name Type Description
data Object | Array

data returned from the API. For object endpoints (e.g., "/user/:userid"), data is an Object, but for collection endpoints (e.g., "/documents"), data is an Array of Objects
request Object

contains information about the request made
timestamp string

timestamp of the request
responsetime number

amount of time the API call took in seconds
etag string

ETag for request. Please refer to our documentation here for more information on ETags

The ListenerResultCallback handles results from the Speech Recognition API. A ListenerResultCallback should at minimum handle the result parameter.

Name Type Description
result ListenerResult

result object containing speech recognition result
resultIndex number

the index of the provided result in the results array
results Array

an array of ListenerResult objects received during the current speech recognition session
event Event

the original event received from the underlying SpeechRecognition instance

QueryParametersObject

A QueryParameter Object has one or more fields that allow you to narrow down the list of items returned from a collection. A QueryParameter object looks like the following:

Properties:
Name Type Argument Description
query string <optional>

search query string to retrieve specific objects that match the query. See the documentation on search query syntax for more information
start number <optional>

The index of the first object in the returned list of objects. This can be used for paging through large collections of objects.
limit number <optional>

The maximum number of individual objects to be returned in the response. If not specified, the default is 10. The maximum allowed value is 50.
since number | string <optional>

A Unix timestamp or strtotime date value that specifies the start of a range of time-based data. Only objects created after this timestamp will be returned in the response.
until number | string <optional>

A Unix timestamp or strtotime date value that specifies the end of a range of time-based data. Only objects created before this timestamp will be returned in the response.
Example
var queryParams = {
     query: "san francisco", // return items that match the string 'san francisco'
     start: 4,   // return items starting at the 4th index
     limit: 10,  // limit the number of returned items to 10
     since: "last Monday",   // return items created since last Monday
     until: "yesterday"     // return items that were crated before yesterday
}
